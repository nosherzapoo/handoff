# handoff

Handover repository for work being transitioned to a new maintainer.

## Projects

### `osb-operator-workbook/`

Builds a single Excel workbook tracking the **online sports-betting market share** of the
major US operators (FanDuel, DraftKings, BetMGM, Fanatics, Caesars, ESPN Bet, Others) by
Handle and GGR, month by month, across the 20 states that publish operator-level online
numbers. It pulls the numbers **straight from the live OSBdata API** (`api.osbdata.com`) —
no raw CSVs or tracker codebase required — and writes a 14-sheet, formula-driven
`.xlsx` (Handle, GGR, Hold, Like-for-Like Y/Y, Quarterly Trends, by-state panels, Coverage,
Sources, Methodology).

Full guide: [`osb-operator-workbook/README.md`](osb-operator-workbook/README.md).

Quick start (run from that folder):

```bash
pip install -r requirements.txt
python3 build_operator_excel_from_api.py    # → US_Operator_TimeSeries_Online.xlsx
```

### `ny-scraper/`

Automated pipeline that tracks New York's online sports-betting market. It downloads the
weekly reports the NY State Gaming Commission publishes for all 9 licensed operators
(each as a PDF and an Excel), extracts Handle and GGR per week into `ny_gaming_data.csv`,
builds a formatted weekly exhibit, detects changes versus the previous run, and emails an
update with the spreadsheets attached. Runs unattended via GitHub Actions.

Full guide: [`ny-scraper/README.md`](ny-scraper/README.md) — **read section 3 first**, it
explains that the recurring schedule is driven by an external cron-job.org trigger, not by
GitHub.

### `pollstar/`

Tooling to extract the Pollstar Data Cloud Boxoffice dataset into a flat CSV. Pollstar has
no export button and no public API, so this calls the internal `boxoffice2` API, decrypts
the AES-encrypted response, and writes every engagement (EventDate on or after 2010-01-01)
to `concerts.csv`.

Full guide: [`pollstar/README.md`](pollstar/README.md).

Quick start (run from this repository root):

```bash
# 1. Put a valid Pollstar bearer token in ./jwt.txt (see pollstar/README.md, section 3).
# 2. Pull the data:
./pollstar/scripts/refresh_data.sh
```

For both projects only **code and docs** are tracked; the datasets and caches they
generate are gitignored and regenerated by running the pipelines. The Pollstar token and
extracted data live at the repository root; the downstream analysis scripts are kept
separately and are not part of this repository.
